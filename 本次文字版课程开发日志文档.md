## 3. 关键勘误 ①：`ruff check` 抓不到格式

- 笔记原话：门 = `pytest -q && ruff check src/`，并称「ruff 报格式问题」。
- **实测**：`ruff check src/` 对 `def add(a,b)` / `a+b` 这类空格输出 `All checks passed!`。
- 原因：这是「格式（format）」问题，归 `ruff format` 管；`E225/E231` 在 ruff 0.15 是 *preview* 规则默认不启用。
- **修正**：所有门改为 `pytest -q && ruff check src/ && ruff format --check src/`。
  - 行为门 = `pytest`；lint 门 = `ruff check`；**格式门 = `ruff format --check`**。
- 实测三段退出码：`pytest=1`（除零）、`ruff check=0`、`ruff format --check=1`（格式被抓到）。

> 这处错误已从笔记**删除并改正**（含第 2 课门、Step 2、SKILL.md、Step 5 loop.sh、第 4 课三道闸表述）。

## 5. 勘误 ②：核实 Claude Code 能力性断言（claude-code-guide 子 agent）

对第 0/1/2/4 课里 14 条关于 Claude Code 的断言逐条核实，结果：**11 条正确，2 条错、1 条不全**。

| 项 | 原表述 | 判定 | 改正 |

| `/loop` 过期 | 3 天 | ❌ | **7 天**（可 `--resume`/`--continue` 恢复） |

`--allowedTools "Bash(pytest*)"` 通配写法、`--permission-mode acceptEdits`（另有 default/plan/auto/dontAsk/bypassPermissions）、
`git worktree`+`--worktree`+`isolation: worktree`、Skills 路径、`.claude/agents/*`、`SubagentStop` hook、
`anthropics/claude-code-action@v1`。

> 以上 3 处已在**笔记 + 课程文件**中改正（3 天→7 天、删 `--max-budget-usd`、连接器补全）。
