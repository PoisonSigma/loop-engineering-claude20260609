## ⚠️ 关键发现（对原笔记的勘误）

笔记说「ruff 报格式问题」，但**实测 `ruff check src/` 输出 `All checks passed!`**。

原因：`calc.py` 里 `def add(a,b)` / `a+b` 是**格式问题**，归 `ruff format` 管；
而 `E225/E231` 这类空格规则在 ruff 0.15 里是 *preview* 规则，默认不启用：

```text
$ ruff check --select E231,E225,E226 src/
warning: Selection `E231` has no effect because preview is not enabled.
All checks passed!
```

要让「格式门」真的有信号，得用 `ruff format --check`：

```text
$ ruff format --check src/
Would reformat: src/calc.py
1 file would be reformatted
```

退出码：`1`（格式门有效 ✅）

## 结论

✅ Step 2 跑通，而且**意外验证了第 4 课「门会腐烂」**：
照抄笔记的 `ruff check src/` 会得到一个**假绿的格式门**。
本课后续统一把门修正为 `pytest -q && ruff check src/ && ruff format --check src/`。
