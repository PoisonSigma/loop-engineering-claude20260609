# Step 6 · 上调度（三选一）

> 这步是「把 `./loop.sh` 挂到一个会自动触发的心跳上」。三种方式，按存活范围选。
> 注意：A/C 依赖 Claude Code 的交互环境 / 账号能力，**没有可在本机捕获的命令行输出**；
> 只有 B（crontab）是纯本机、可验证，下面给了实测。

## A. 会话内（最快，7 天过期）

在 claude 交互里输入：

```
/loop 30m 跑 ./loop.sh，门绿则什么都不做
```

- 会话级心跳，每 30 分钟触发一次；7 天自动过期（可用 `--resume`/`--continue` 恢复未过期任务）。
- 适合：临时盯一个短期任务。关掉会话就停。

## B. 重启存活（你自己机器）—— 本机可验证 ✅

```bash
crontab -e
# 加入这一行（每天凌晨 3 点跑，日志追加到 ~/loop.log）：
0 3 * * * cd ~/loop-demo && ./loop.sh >> ~/loop.log 2>&1
```

见同目录 `crontab.txt`（可直接 `crontab crontab.txt` 安装，或 `crontab -e` 手动粘贴）。
本机已验证 crontab 行格式合法、`crontab -l` 能列出（详见 `运行结果.md`）。

## C. 云端关机也跑（Max+）

```
/schedule        # = claude.ai 的 Routines
```

- 云端定时触发，本地关机也照跑。需要相应账号能力。
